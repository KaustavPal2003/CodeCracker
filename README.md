# CodeCracker 🔥

A real-time competitive programming stat tracker that aggregates your ratings and contest history from **Codeforces**, **LeetCode**, **CodeChef**, and **AtCoder** into a single dashboard.

🔗 **Live:** [codecracker-odbi.onrender.com](https://codecracker-odbi.onrender.com)

---

## Features

- **Unified dashboard** — view all four platform ratings and solved counts in one place
- **Real-time updates** — stats pushed via WebSocket (Django Channels + Redis), no polling
- **Rating history charts** — contest-by-contest rating progression across all platforms
- **Head-to-head comparison** — compare your stats against any other registered user
- **Leaderboard** — weighted composite score (CF×0.35 + CC×0.25 + AC×0.25 + LC×0.15)
- **Per-user platform handles** — your login username can differ from your CP handles
- **Export** — download your full contest history as CSV or PDF
- **RBAC** — three roles: user, moderator, admin
- **Email verification** — HMAC-based token with 24h TTL
- **Dark/light theme** — persistent theme switcher

## Tech Stack

| Layer | Technology |
|---|---|
| Backend | Django 4.2, Daphne (ASGI) |
| Real-time | Django Channels, Redis (channel layer) |
| Auth DB | PostgreSQL (via dj-database-url) |
| Stats DB | MongoDB Atlas (via MongoEngine) |
| Caching | Redis (24h rating history cache) |
| Frontend | Vanilla JS, Chart.js, WebSocket API |
| Static files | WhiteNoise (compressed) |
| Deployment | Docker, Render |
| Security | django-csp, django-ratelimit (60 req/min per WS) |

## Architecture

```
Browser
  │
  ├── HTTP  ──► Daphne (ASGI) ──► Django views
  │                                    │
  └── WS ────► Django Channels         ├── PostgreSQL (users, sessions)
                    │                  └── MongoDB Atlas (stats, history)
               Redis channel layer
                    │
               StatsConsumer / LeaderboardConsumer
```

**Key decisions:**
- ASGI over WSGI — needed for WebSocket support alongside HTTP
- Dual database — PostgreSQL for auth (ACID), MongoDB for stats (flexible schema per platform)
- Redis dual role — Celery broker + Django cache in one service
- Lazy MongoDB connection — skips connection during `collectstatic` and `migrate` to keep Docker builds clean
- zlib compression on all WebSocket frames — binary payloads for large rating history arrays

## Local Development

### Prerequisites
- Python 3.11+
- Docker & Docker Compose
- MongoDB Atlas account (free tier)

### Setup

```bash
git clone https://github.com/KaustavPal2003/CodeCracker.git
cd CodeCracker
```

Create a `.env` file in the project root:

```env
DJANGO_SECRET_KEY=your-secret-key
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost
MONGO_USERNAME=your-mongo-username
MONGO_PASSWORD=your-mongo-password
REDIS_URL=redis://localhost:6379
DATABASE_URL=sqlite:///db.sqlite3
APP_DOMAIN=127.0.0.1:8000
```

Install dependencies and run:

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

Or with Docker:

```bash
docker build -t codecracker .
docker run -p 8000:8000 --env-file .env codecracker
```

## Deployment (Render)

The project includes a `render.yaml` Blueprint that provisions:
- Web service (Docker, free tier)
- PostgreSQL database
- Redis (Key Value store)

Deploy in one click:

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy)

**Required environment variables** (set manually in Render dashboard):

| Variable | Description |
|---|---|
| `MONGO_USERNAME` | MongoDB Atlas database username |
| `MONGO_PASSWORD` | MongoDB Atlas database password |
| `DJANGO_SECRET_KEY` | Auto-generated by Render Blueprint |
| `ALLOWED_HOSTS` | Your Render service URL |
| `APP_DOMAIN` | Same as ALLOWED_HOSTS (for CSP/WebSocket) |

> **Note:** Render free tier spins down after 15 minutes of inactivity. First request after spindown takes ~30 seconds.

## Security

- HMAC email verification tokens (SHA-256, 24h TTL in Redis)
- Content Security Policy via `django-csp`
- WebSocket rate limiting — 60 requests/minute per username via `django-ratelimit`
- CSRF protection on all state-changing endpoints
- Secure cookies (`SESSION_COOKIE_SECURE`, `CSRF_COOKIE_SECURE`)
- All credentials via environment variables — never hardcoded
